01 / Premise

Promises are soft. Stakes are not.

Every relationship runs on promises — the unspoken kind and the written kind. They erode quietly. Nobody signs them. Nobody enforces them. When they break, both parties debate what was ever agreed in the first place.

LoveClaw offers a different posture: two partners define their own rules, stake collateral against them, and let a neutral system do the remembering.

The goal is not punishment. It is legibility — of what was promised, and what was kept.

02 / Architecture

Four layers, one commitment.

A locally-private agent runs on each partner's device, coordinating through an encrypted relay. Rules are evaluated mutually; enforcement is delegated to a smart contract holding shared collateral.

03 / Components

The four moving parts.

01 · client

The local agent.

A privacy-first daemon running on each partner's phone. It observes only the signals both partners explicitly agreed to share — app categories, location fences, presence. Raw data never leaves the device.

02 · transport

The secure relay.

An end-to-end encrypted channel that couriers signed messages between agents. The relay sees timing and size — never content. Messages are ephemeral by default; a hash chain is anchored for auditability.

03 · logic

The rule engine.

Both partners co-author the rules in plain language, which compile to predicates. Example: if(late_hours > 2 && ¬disclosed) → breach. Evaluation is mutual — a breach requires both agents to agree.

04 · settlement

The smart contract.

A shared vault holding collateral — cash, crypto, or a charity pledge. On a confirmed breach, funds route according to pre-agreed terms: partial forfeit, donation, or cool-off lockup. On graduation, collateral returns in full.

04 / Lifecycle

One breach, from signal to settlement.

T+00s

Signal observed

Local agent on Phone A registers an event matching a monitored predicate.

T+02s

Cross-check

Encrypted request to Phone B's agent confirms independent context.

T+04s

Rule triggers

Predicate evaluates true on both sides. Breach state is ratified.

T+06s

Contract called

Signed attestation submitted on-chain. A grace window opens for challenge.

T+72h

Settlement

If uncontested, collateral routes per agreement. Both partners notified.

05 / Principles

What LoveClaw refuses to do.

The protocol is opinionated. It exists to make commitments legible — not to surveil, coerce, or punish. These guardrails are non-negotiable.

001 · Consent

No rule without both signatures.

Every predicate is co-authored and co-signed. Either partner can veto. Unilateral monitoring is not a feature.

002 · Locality

Raw signals never leave the device.

Only attestations cross the wire. Location traces, messages, and app contents remain on-device at all times.

003 · Exit

Graceful dissolution is a first-class action.

Ending the protocol returns all collateral with no penalty. Leaving a relationship is never a breach.
